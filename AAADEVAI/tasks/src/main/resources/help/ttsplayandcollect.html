Text To Speech, Play And Collect Help

Definición de la tarea

La tarea Reproducir y recolectar reproduce un anuncio y recopila dígitos de multifrecuencia de tono dual (DTMF, por sus siglas en inglés) presionados por la parte llamante o llamada.

Play and Collect es una tarea asíncrona. Se reanuda cuando el anuncio expira, finaliza o se interrumpe por los dígitos que se presionan.

Propiedades de la tarea

En la siguiente lista de propiedades, la fuente de medios es obligatoria. La fuente URI / texto de los medios tiene prioridad sobre los valores seleccionados para el espacio de nombre de los medios, el nombre del grupo de medios y el archivo de medios.

Los valores especificados por la asignación de entrada tienen prioridad sobre los valores correspondientes ingresados ​​en la ventana Propiedades.

  • Media URI / Texto (obligatorio): La fuente que identifica la lista de recursos de audio o la cadena de texto a voz (TTS) que se reproducirá en la llamada. Valor legal para la fuente: una lista de cstore: // o file: // o http: // URI para recursos de audio o cadenas de texto TTS para sintetizar en voz.

  • Locale regional: lenguaje de solicitud admitido por Avaya Aura> Media Server (MS).

  • Number of digits: El número de dígitos que deben ser recogidos. Entrada válida: número mayor o igual a 1.

  • Termination key: La clave de terminación que detiene una operación de recolección de dígitos. Un valor posible podría ser un dígito '0-9', '*' o '#' símbolo. El usuario puede ingresar la clave de terminación a voluntad para terminar el anuncio. El valor predeterminado es el símbolo #. Nunca se le pedirá al usuario que presione la tecla de terminación. Sin embargo, el usuario puede optar por finalizar el proceso presionando la tecla de terminación. El momentAvaya Aura Media Server (MS) detecta el último dígito del número requerido de dígitos ingresados ​​por el usuario, la tarea recopila y devuelve la salida. No es necesario presionar la tecla de terminación.

  • Handle: El número / dirección para la cual se reproducirá el anuncio y los dígitos recopilados.

  • Collect digits from: El destinatario del anuncio, la parte llamada o llamante. Entradas válidas: llamada y llamada.

  • Interrupt announcement (optional): Determina si el anuncio de audio se interrumpe cuando se presiona un dígito. Verdadero (predeterminado) interrumpe el anuncio. Falso no lo hace.

  • Timeout (optional): Número de segundos a esperar para recibir el primer dígito. El valor predeterminado es 60 segundos. Si no se recibe un dígito en el tiempo especificado, la WFD continúa con la siguiente tarea.

  • Cloud Provider- Opción para seleccioner el tipo de proveedor para realizar Text To Speech
    • Google
    • IBM
  • Select The Language - Opción para seleccioner el idióma de para realizar Text To Speech
    • es - Español
    • en - Ingles
    • pt - Portugües

Si se proporcionan los dígitos Handle y Collect from, entonces Handle tiene prioridad. Si no hay ninguno, se utiliza la parte que llama.

Todos los campos obligatorios deben asignarse mediante la ventana de asignación de entrada o ingresarse en la ventana Propiedades para que la validación tenga éxito. Los dígitos Handle o Collect deben tener un valor.

Input schema

  • ucid (Universal Call Identifier) (mandatory) - Identificador de llamada de la llamada donde se reproduce el anuncio. Asigne el ucid desde el appid del evento Call o desde el esquema de salida de la tarea Make Call.
  • mediauri - Texto de entrada para crear el audio que se reproducirá en la llamda o URL para obtener el archivo de audio
  • handle - El número / dirección para quien se reproducirá el anuncio.
  • participant - El destinatario del anuncio: la parte llamada o la parte que llama. Entradas válidas: callid, llamada, llamada.
  • interruptibility (optional) - Determina si el anuncio de audio se interrumpe cuando se presiona un dígito. Verdadero (predeterminado), interrumpe el anuncio. Falso no lo hace.
  • duration - La duración de una solicitud de llamada en segundos.
  • iterateCount - El número de veces que se reproduce el anuncio.
  • announcementForever (Optional) - Configura los medios para que se reproduzcan para siempre sin ser interrumpidos por el vencimiento de un temporizador. Los parámetros de duración y recuento de iteraciones no se utilizan cuando este indicador se establece en verdadero. El valor predeterminado es falso.
  • locale - Lenguaje de solicitud admitido por Avaya Aura® Media Server (MS).
  • useLocale - Se aceptan valores verdaderos y falsos. Si es verdadero, la asignación / propiedad de la configuración regional se utiliza para reproducir medios. Si no se proporciona, se utiliza el atributo de configuración regional predeterminado de System Manager.
  • language - Opción para seleccioner el idióma de para realizar Text To Speech
  • cloudProvider- Opción para seleccioner el tipo de proveedor para realizar Text To Speech

Si se asignan los dígitos de Handle y Collect from, entonces Handle tiene prioridad. Si no hay ninguno, se utiliza la parte que llama.

Output schema

  • ucid: ID de llamada universal de la llamada.

  • digits: Dígitos recogidos por la tarea.

  • status:

    • FAILED - Si la colección de dígitos falló antes de que se cumpliera algún criterio de finalización.

    • NUMBER_OF_DIGITS - Si la colección terminó y se cumple el número de dígitos.

    • STOPPED - La operación de recolección de dígitos fue detenida explícitamente por el servicio de cliente.

    • TERMINATION_KEY - Si la colección terminó y se cumplió la clave de terminación.

    • TIMEOUT - Si la colección se completa y no se ha introducido DTMF.

    Busque los siguientes mensajes en el estado cuando una tarea no es exitosa, correlacione la salida y agregue la lógica de manera apropiada:

    • shortReason:'AFTER_ANSWER

    • shortReason:' MEDIA_SERVER_FAILURE

    • 'shortReason':'AFTER_ANSWER'

Casos de uso

La conversión texto-voz es la generación por medios automáticos de una voz artificial que genera el sonido producido por una persona al leer un texto cualquiera en voz alta. o una voz artificial. Es decir, son sistemas que permiten la conversión de textos en voz sintética. Los conversores de texto-voz son conocidos también con las siglas CTV o por las siglas en inglés TTS (Text To Speech).
